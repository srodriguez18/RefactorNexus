# ADR-002: Stack tecnológico

## Contexto
El legacy estaba construido en Python/Flask + JavaScript vanilla + SQLite.
La decisión de migrar a TypeScript fullstack ya estaba tomada por la
dirección. Había que elegir el framework HTTP, el ORM, la herramienta
de build del frontend y la base de datos de destino.

## Opciones consideradas

**Backend — Fastify vs Express**
Express es el framework más veterano con más recursos en internet.
Fastify es más moderno, notablemente más rápido, fue diseñado desde
el inicio pensando en TypeScript y tiene validación de esquemas
integrada. Para un sistema nuevo con TypeScript estricto, Fastify
es la elección más coherente.

**ORM — Prisma vs Drizzle vs TypeORM**
TypeORM tiene tipado débil y es propenso a errores en runtime.
Drizzle es muy nuevo con ecosistema limitado. Prisma tiene el mejor
DX con TypeScript, genera tipos automáticamente desde el schema,
y su Prisma Studio facilita verificar migraciones. La migración de
datos del legacy se verificó visualmente con Prisma Studio.

**Frontend build — Vite + React vs Next.js**
Next.js agrega complejidad de SSR que no necesita este sistema interno.
Vite arranca en menos de un segundo, usa módulos ES nativos y es el
estándar actual para SPAs con React. Para un ERP interno sin necesidades
de SEO, Vite es suficiente y más simple.

**Base de datos — SQLite vs PostgreSQL**
El legacy usaba SQLite. Para el sprint 1 de modernización mantener
SQLite reduce fricción — misma base de datos, sin servidor adicional
que levantar. El schema de Prisma permite migrar a PostgreSQL cambiando
una línea en el datasource cuando el sistema lo requiera.

## Decisión
Fastify + Prisma + SQLite + Vite + React, todo en TypeScript estricto.

## Consecuencias

**Positivas:**
- Tipos generados automáticamente por Prisma eliminan errores de
  mapeo entre base de datos y código
- Fastify + TypeScript da autocompletado completo en handlers y schemas
- Vite hace el desarrollo frontend significativamente más ágido
- SQLite sin servidor simplifica el arranque en máquina limpia
- Un solo lenguaje (TypeScript) en todo el stack reduce el cambio
  de contexto mental

**Negativas:**
- Fastify tiene menos ejemplos y recursos que Express — algunos
  patrones son menos obvios al inicio
- SQLite tiene limitaciones de concurrencia para producción con
  múltiples escrituras simultáneas — migrar a PostgreSQL será
  necesario cuando el sistema escale
- Prisma agrega un paso de generación de cliente que hay que recordar
  correr después de cambios al schema
